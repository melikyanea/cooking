# ТЗ: Приложение для планирования питания

> Версия 1.1 · Дата: 2026-06-06

---

## 1. Описание продукта

Веб-приложение для людей, которым сложно самостоятельно составить сбалансированный рацион и сформировать список покупок. Пользователь за 2 минуты отвечает на несколько вопросов и получает готовое меню на 1–7 дней с полным списком ингредиентов.

**Тип приложения:** SPA с вызовами Claude API для генерации меню. Деплой на Vercel/Netlify.

---

## 2. Где используется модель, а где нет

| Этап | Claude API | Причина |
|------|:----------:|---------|
| Генерация меню по параметрам квиза | **Да** | Учитывает баланс КБЖУ, разнообразие, пожелания в свободной форме, любые исключения |
| Замена блюда / всей подборки | **Да** | Модель знает контекст текущего рациона и не дублирует уже стоящие блюда |
| Агрегация ингредиентов | Нет | Детерминированная математика над данными из ответа модели |
| Разбивка на закупки | Нет | Справочник категорий продуктов (статика) |
| Чекбоксы покупок | Нет | localStorage |
| Экспорт в файл | Нет | Форматирование строк → Blob |

## 5. Модели данных

### 5.1 TypeScript интерфейсы

```typescript
type MealType = 'breakfast' | 'lunch' | 'dinner';
type Difficulty = 'easy' | 'medium' | 'hard';
type MeasureUnit = 'г' | 'мл' | 'шт' | 'ст.л.' | 'ч.л.' | 'щепотка' | 'по вкусу';
type IngredientCategory = 'perishable' | 'storable';

interface Ingredient {
  id: string;               // slug: 'chicken_breast'
  name: string;             // 'Куриная грудка'
  amount: number;
  unit: MeasureUnit;
  category: IngredientCategory;
}

interface RecipeStep {
  stepNumber: number;
  instruction: string;
}

interface Dish {
  id: string;
  name: string;
  description: string;      // 2–3 предложения
  mealType: MealType;
  cookingTime: number;      // минуты
  difficulty: Difficulty;
  ingredients: Ingredient[];
  steps: RecipeStep[];
}

interface DayPlan {
  dayNumber: number;        // 1, 2, 3…
  breakfast?: Dish;
  lunch?: Dish;
  dinner?: Dish;
}
```

### 5.2 Состояние приложения (Zustand)

```typescript
interface AppState {
  // Квиз
  mealTypes: MealType[];
  duration: 1 | 3 | 7;
  portions: number;          // 1–99
  exclusions: string[];      // ['лук', 'лактоза', ...] — свободные строки
  extraNotes: string;        // пожелания из шага 5

  // Меню (возвращает модель)
  days: DayPlan[];
  generatedAt: string;       // ISO timestamp

  // Покупки (клиентская логика)

interface ShoppingItem {

  totalAmount: number;

  category: IngredientCategory;

```

## 6. Интеграция Claude API

### 6.1 Генерация меню

**Когда вызывается:** после нажатия "Получить меню" на шаге 5 квиза.

**Промпт (системный):**
```
Ты — диетолог и шеф-повар. Составляй сбалансированные рационы питания.
Отвечай строго в формате JSON без лишнего текста.
Соблюдай баланс белков, жиров и углеводов по дням.
Не повторяй одно и то же блюдо в рамках одного рациона.
Каждое блюдо должно содержать рецепт с шагами и точные ингредиенты с граммовками.
```

**Пользовательский промпт (формируется из квиза):**
```
Составь рацион со следующими параметрами:
- Приёмы пищи: ${mealTypes.join(', ')}
- Количество дней: ${duration}
- Порций на приём: ${portions}
- Исключить: ${exclusions.join(', ') || 'ничего'}
- Дополнительно: ${extraNotes || 'нет'}

Верни JSON в формате:
{
  "days": [
    {
      "dayNumber": 1,
      "breakfast": { ...Dish } | null,
      "lunch": { ...Dish } | null,
      "dinner": { ...Dish } | null
    }
  ]
}
```

**Параметры запроса:**
```typescript
const response = await anthropic.messages.create({
  model: 'claude-haiku-4-5-20251001',
  max_tokens: 8000,
  messages: [{ role: 'user', content: userPrompt }],
  system: systemPrompt,
});
```

**После получения ответа:** парсим JSON, валидируем структуру, сохраняем в стор. Если парсинг упал — показываем ошибку с кнопкой "Попробовать снова".

---

### 6.2 Замена одного блюда

**Когда вызывается:** по нажатию "Заменить блюдо" у конкретного приёма пищи.

**Промпт:**
```
Уже есть следующий рацион:
${текущие блюда рациона списком}

Замени блюдо "${currentDish.name}" (${mealType}, день ${dayNumber}).
Требования: ${exclusions}, ${extraNotes}.
Предложи другое блюдо — не повторяй то, что уже есть в рационе.
Верни JSON одного блюда в том же формате Dish.
```

**Параметры:** `max_tokens: 2000` — достаточно для одного блюда.

---

### 6.3 Замена всей подборки

Повторный вызов `generateMenu` с теми же параметрами квиза. Модель каждый раз генерирует новый вариант за счёт температуры.

---

### 6.4 Обработка ошибок API

| Ситуация | Поведение |
|---------|-----------|
| Сеть недоступна | Toast: "Нет подключения к интернету" |
| Таймаут (>15 сек) | Toast: "Сервер не отвечает, попробуйте ещё раз" |
| Невалидный JSON в ответе | Повтор запроса (1 раз автоматически), затем ошибка |
| Модель отказала (refusal) | Сообщение: "Не удалось составить меню с такими ограничениями. Попробуйте изменить параметры." |

## 8. Экраны и UX

### 8.1 Главная страница / Каталог

**Структура:**
- Hero-блок: заголовок + подзаголовок + главная CTA-кнопка **"Составить свой рацион"**
- Раздел **"Завтраки"** — сетка из 12 карточек
- Раздел **"Обеды + Ужины"** — сетка из 12 карточек
- Повторная CTA-кнопка в конце страницы

**Карточка блюда:**
- Название + описание (max 120 символов)
- Кликабельна → открывается рецепт (модалка)

---

### 8.2 Квиз (5 шагов)

**Общие правила:** прогресс-бар "Шаг X из 5", кнопка "Назад" сохраняет ответы, "Далее" неактивна до заполнения обязательных полей.

#### Шаг 1 — Приёмы пищи
- Множественный выбор: **Завтрак / Обед / Ужин**
- Обязательный: хотя бы 1

#### Шаг 2 — Количество дней
- Одиночный выбор: **1 день / 3 дня / Неделя**
- Обязательный

#### Шаг 3 — Количество порций
- Быстрый выбор: **1 / 2 / 4** + "Своё значение" (целое 1–99)
- Обязательный

#### Шаг 4 — Исключения
- Чекбоксы: **Лук / Чеснок / Лактоза**
- Текстовое поле "Другое" (необязательное, max 200 символов)
- Необязательный шаг — "Далее" всегда активна

#### Шаг 5 — Дополнительные пожелания
- Textarea (необязательная, max 300 символов)
- Кнопка **"Получить меню"** → вызов Claude API → spinner "Подбираем меню…"

---

### 8.3 Экран предложенного меню

```
День 1
  Завтрак: [Название] + описание + [Смотреть рецепт] [Заменить блюдо]
  Обед:    [Название] + описание + [Смотреть рецепт] [Заменить блюдо]
  Ужин:    [Название] + описание + [Смотреть рецепт] [Заменить блюдо]
День 2 …
```

**Глобальные действия:**
- **"Заменить всю подборку"** → подтверждение → повторный вызов API
- **"Подтвердить рацион"** → список покупок

**Замена блюда:** строка уходит в загрузку, вызывается `replaceDish`, ответ вставляется с анимацией.

---

### 8.4 Экран списка покупок

**Группы:**
1. **Скоропортящиеся** — мясо, рыба, молочка, зелень
2. **Хранимые** — крупы, масла, специи

**Каждый элемент:** чекбокс + название + количество + единица + пометка повторов ("×3 блюда").

**Счётчик:** "Куплено: 5 из 18" + кнопка "Сбросить отметки".

**Кнопка "Скачать список покупок":** TXT (основной) или PDF. Чекбоксы в файл не попадают.

## 9. Логика клиентской обработки (без модели)

### 9.1 Агрегация ингредиентов

```typescript
function buildShoppingList(days: DayPlan[], portions: number): ShoppingItem[] {
  // 1. Все блюда из меню → все ингредиенты
  // 2. Суммировать одинаковые (по нормализованному имени)
  // 3. Умножить на portions
  // 4. Привести единицы к стандарту (г/кг, мл/л)
  // 5. Округлить до удобных значений
  // 6. Определить категорию по справочнику INGREDIENT_CATEGORIES
  // 7. Заполнить usedInDishes для пометки повторов
}
```

**Округление:** граммы/мл → до 10, штуки → вверх до 1, ложки → до 0.5.

### 9.2 Экспорт TXT

```typescript
function exportTXT(items: ShoppingItem[], meta: ExportMeta): void {
  const lines = [
    `Список покупок — ${meta.duration} дн., ${meta.portions} порц.`,
    `Сформирован: ${new Date().toLocaleDateString('ru-RU')}`,
    '', '=== СКОРОПОРТЯЩИЕСЯ ===',
    ...items.filter(i => i.category === 'perishable')
            .map(i => `[ ] ${i.name} — ${i.totalAmount} ${i.unit}`),
    '', '=== ХРАНИМЫЕ ===',
    ...items.filter(i => i.category === 'storable')
            .map(i => `[ ] ${i.name} — ${i.totalAmount} ${i.unit}`),
  ];
  const blob = new Blob([lines.join('\n')], { type: 'text/plain;charset=utf-8' });
  const a = Object.assign(document.createElement('a'), {
    href: URL.createObjectURL(blob),
    download: `shopping-list-${Date.now()}.txt`,
  });
  a.click();
}
```

## 10. Персистентность

Один ключ `meal-planner-storage` в localStorage через Zustand persist.

**Что сохраняется:** параметры квиза, сгенерированное меню, отмеченные чекбоксы.

**Инвалидация:** если `generatedAt` старше `duration + 7 дней` → автосброс меню при старте.

## 12. Граничные случаи

| Ситуация | Поведение |
|---------|-----------|
| Пользователь открыл /menu без квиза | Редирект на главную |
| API вернул невалидный JSON | Автоповтор 1 раз, затем ошибка |
| Модель не может учесть все ограничения | Сообщение с предложением ослабить параметры |
| Ингредиент в разных единицах из разных блюд | Привести к граммам/мл при агрегации |
| Ингредиент без количества ("соль") | Показывать "по вкусу" |
| Все чекбоксы отмечены | "Всё куплено! Приятного приготовления" |
